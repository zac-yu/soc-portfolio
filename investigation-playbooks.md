# SOC Investigation Playbooks

## Playbook 1 — Repeated failed Windows logons

1. Identify the user, endpoint, source IP, logon type and failure reason.
2. Compare failures with successful Event ID `4624` activity.
3. Determine whether the pattern is user error, a stale service credential, password spraying or brute force.
4. Review the source across other users and hosts.
5. Escalate when failures are distributed, followed by success, or linked to privileged accounts.
6. Recommend password reset, account lock, source isolation or rule tuning as appropriate.

## Playbook 2 — Suspicious PowerShell

1. Review the complete command line and decode Base64 content safely if present.
2. Inspect parent and child processes.
3. Identify downloaded files, URLs, DNS lookups and outbound connections.
4. Check file hashes and endpoint detections.
5. Search for persistence such as scheduled tasks or registry run keys.
6. Isolate the endpoint if execution is unauthorized and evidence indicates compromise.

## Playbook 3 — Suricata IDS alert

1. Record signature ID, category, severity, action and timestamp.
2. Validate source/destination IPs, ports and direction.
3. Confirm whether the traffic was generated by an approved lab test.
4. Pivot to surrounding network events and endpoint processes.
5. Determine whether the signature is a true positive, benign true positive or false positive.
6. Document containment or tuning recommendations.

## Playbook 4 — Suspicious Microsoft 365 sign-in

1. Check user, application, IP, country, device, authentication method and result.
2. Compare with the user's normal locations and recent successful sign-ins.
3. Review MFA, Conditional Access, impossible travel and risky sign-in indicators.
4. Check mailbox rules, forwarding, consent grants and unusual activity.
5. Revoke sessions and reset credentials if compromise is suspected.
6. Preserve the timeline and escalate based on account privilege and impact.
